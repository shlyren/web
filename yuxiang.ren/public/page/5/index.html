    runtime

    2016-03-26
    0 Discuss

    runtime简介

    1. Runtime简称运行时,OC就是运行时机制,也就是在运行时候的一些机制,其中最主要的是消息机制.
    2. 对于C语言,函数的调用在编译的时候回决定调用哪个函数.
    3. 对于OC的函数,属于动态调用过程,在编译的时候并不能决定真正调用哪个函数,只要在真正运行的时候才会根据函数的名称找到对应的函数来调用.
    4. 事实证明:
    • 在编译阶段,CO可以调用任何函数,即使这个函数未实现,只要声明过就不会报错.
    • 在编译阶段,C语言调用未实现的函数就会报错.

    const,static,extern简介

    2016-03-26
    0 Discuss

    const与宏的区别(面试题):

    1. const简介:之前常用的字符串常量,一般是抽成宏,但是苹果不推荐我们抽成宏,推荐我们使用const常量.
      主要是因为以下四点:
      • 编译时刻:宏是预编译,const是编译阶段.
        • 编译检查:宏不做检查,不会报编译错误,只是替换,const会编译检查,会报编译错误.
        • 宏的好处:宏能定义一些函数,方法,const不能.
        • 宏的坏处:使用大量红,容易造成编译时间久,每次都需要重新替换.
